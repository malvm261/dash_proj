# 🔧 Диагностика проблем

## ❌ "localhost:5000 не открывается"

### Шаг 1: Проверьте вывод консоли сервера

Запустите в `PiServer`:
```bash
dotnet run
```

**Должно быть:**
```
╔══════════════════════════════════════════════════════╗
║   Сервер распределённого вычисления числа π          ║
╚══════════════════════════════════════════════════════╝
Веб-интерфейс: http://localhost:5000
API:           http://0.0.0.0:5000/api/*
...
```

**Если видите ошибку:**
- Скопируйте текст ошибки
- Проверьте, нет ли других процессов на порту 5000

### Шаг 2: Проверьте, занят ли порт 5000

**На Windows:**
```powershell
netstat -ano | findstr :5000
```

**На Mac/Linux:**
```bash
lsof -i :5000
```

Если есть процесс - kill его или используйте другой порт.

### Шаг 3: Проверьте файлы

Файлы должны быть в:
```
PiServer/wwwroot/
├── index.html
├── style.css
└── client.js
```

Проверьте наличие:
```bash
ls -la PiServer/wwwroot/
```

Должны быть все 3 файла ✅

### Шаг 4: Очистите и пересоберите

```bash
cd PiServer
dotnet clean
dotnet build
dotnet run
```

### Шаг 5: Проверьте браузер

1. Откройте: `http://localhost:5000`
2. Проверьте консоль браузера (F12 → Console)
3. Ищите ошибки загрузки

## 📋 Прямая диагностика

### Тест 1: Доступны ли статические файлы?

```bash
# В браузере откройте:
http://localhost:5000/index.html  # Должен загрузиться
http://localhost:5000/style.css   # Должен загрузиться
http://localhost:5000/client.js   # Должен загрузиться
```

### Тест 2: Доступен ли API?

```bash
# В браузере откройте:
http://localhost:5000/api/status  # Должен вернуть JSON
```

Должно быть:
```json
{
  "totalChunks": 100,
  "completedChunks": 0,
  "progress": 0.0,
  "pi1": 0.0,
  "pi2": 3.0,
  "isComplete": false
}
```

### Тест 3: Работает ли консоль браузера?

1. Нажмите **F12** в браузере
2. Откройте вкладку **Console**
3. Должны ли быть ошибки?

Если видите:
- `Failed to fetch` → сервер не запущен
- `404` → файл не найден
- Пусто ✅ → нормально

## 🔴 Типичные ошибки

### Ошибка 1: "Cannot find module"
```
System.IO.FileNotFoundException: wwwroot/index.html not found
```

**Решение:**
- Убедитесь, что файлы в `PiServer/wwwroot/`
- Пересоберите: `dotnet build`

### Ошибка 2: "Port 5000 is already in use"
```
System.Net.Sockets.SocketException: Address already in use
```

**Решение:**
```bash
# Найти процесс
netstat -ano | findstr :5000

# Kill процесс (Windows)
taskkill /PID <PID> /F

# Kill процесс (Mac/Linux)
kill -9 <PID>
```

### Ошибка 3: "Compiler Error"
```
error CS...
```

**Решение:**
```bash
cd PiServer
dotnet clean
dotnet restore
dotnet build
```

### Ошибка 4: "The current directory is not a project"
```
The project file could not be loaded.
```

**Решение:**
```bash
# Убедитесь, что вы в правильной директории
cd PiServer
dotnet run
```

## 🟢 Полная процедура с нуля

Если ничего не работает, начните заново:

```bash
# 1. Перейдите в PiServer
cd PiServer

# 2. Очистите
dotnet clean
rm -rf bin obj

# 3. Восстановите зависимости
dotnet restore

# 4. Постройте
dotnet build

# 5. Запустите
dotnet run

# 6. В браузере откройте
# http://localhost:5000
```

Если вывод:
```
warn: Microsoft.Hosting.Lifetime[0]
      Listening on http://0.0.0.0:5000
```

✅ **Сервер запущен!**

## 🧪 Минималистичный тест

Если вообще ничего не работает, проверьте, запускается ли голый ASP.NET:

```bash
cd PiServer
dotnet run --no-restore
```

Должна быть строка:
```
Listening on http://0.0.0.0:5000
```

Если её нет - проблема с .NET установкой.
