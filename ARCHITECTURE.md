## Поток данных

### Вычисления (слева направо)

```
[Сервер распределяет]
        ↓
 Блок 1..10M итераций
        ↓
[Клиент 1 вычисляет]
        ↓
Серия Лейбница (частичная сумма)
Серия Нилаканта (частичная сумма)
        ↓
[Отправить результат на сервер]
        ↓
[Сервер накапливает]
        ↓
Общая сумма Лейбница
Общая сумма Нилаканта
        ↓
[Вычислить π]
        ↓
π ≈ 4 * сумма_Лейбниц
π ≈ 3 + сумма_Нилаканта
        ↓
[Отправить в браузер]
        ↓
[График обновляется в реал-тайме]
```

### Запрос статуса из браузера (каждые 1 сек)

   ↓

## Алгоритмы

### Ряд Лейбница
```
π/4 = 1 - 1/3 + 1/5 - 1/7 + 1/9 - ...
π/4 = Σ (-1)^k / (2k+1), k = 0, 1, 2, ...

Реализация:
for i in range(start, start+count):
    term = (-1)^i / (2*i + 1)
    sum += term

π ≈ 4 * sum
```

### Ряд Нилаканты
```
π = 3 + 4/(2·3·4) - 4/(4·5·6) + 4/(6·7·8) - ...
π = 3 + Σ (-1)^(k+1) * 4 / ((2k)(2k+1)(2k+2)), k = 1, 2, 3, ...

Реализация:
for i in range(start, start+count):
    k = i + 1
    term = 4 / ((2*k) * (2*k+1) * (2*k+2))
    sum += (-1)^(k+1) * term

π ≈ 3 + sum
```

## Синхронизация

### На сервере (Coordinator)
```csharp
private readonly object _lock = new();  // Глобальный lock

// Все публичные методы используют:
lock (_lock) {
    // Критическая секция
    // Доступ к _totalIterations, _nextStart, _leibnizTotal и т.д.
}
```

Простой, но безопасный подход. При большом количестве клиентов может быть узким местом.

### На клиенте (вычисления)
```csharp
Parallel.For(0, count,
    () => 0.0,                    // инициализация локальной переменной
    (i, _, local) => {
        // Вычисления без lock'а
        return local + term;       // обновление локальной копии
    },
    local => {
        lock (lk) leibniz += local; // merge локальных результатов
    });
```

Оптимально: вычисления без синхронизации, merge только в конце.

## Масштабируемость

### Текущие ограничения

1. **Lock'и на сервере** (~50 клиентов максимум)
   - Каждый запрос получить задачу → блокирует
   - Каждый результат → блокирует
   - При 50+ клиентов есть очередь на lock

2. **Polling вместо WebSocket** (~100 запросов/сек)
   - Браузер спрашивает каждые 1 сек
   - Неэффективно для сотен клиентов

3. **Память на сервере**
   - Каждый клиент → запись в Dictionary
   - Нет очистки при отключении

### Возможные оптимизации

1. **async/await вместо lock'ов**
   ```csharp
   private readonly SemaphoreSlim _semaphore = new(1, 1);
   await _semaphore.WaitAsync();
   try { /* критическая секция */ }
   finally { _semaphore.Release(); }
   ```

2. **WebSocket для браузера**
   ```javascript
   const ws = new WebSocket('ws://localhost:5000/ws');
   ws.onmessage = (e) => updateUI(e.data);
   ```

3. **Heartbeat для отслеживания клиентов**
   ```
   Если клиент не отправлял результаты 30 сек → удалить из списка
   ```
