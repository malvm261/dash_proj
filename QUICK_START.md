# 🚀 Быстрый старт

## За 3 шага запустить всё

### Шаг 1️⃣: Запустить сервер
```bash
cd PiServer
dotnet run
```
Ждите вывода:
```
Веб-интерфейс: http://localhost:5000
```

### Шаг 2️⃣: Открыть браузер
Откройте в любом браузере (Chrome, Firefox, Safari):
```
http://localhost:5000
```

Вы увидите красивый дашборд с графиком!

### Шаг 3️⃣: Запустить клиента (в новом терминале)
```bash
cd PiClient
dotnet run
```

В окне WinForms введите:
```
URL: http://localhost:5000
ID: Client-1
```
Нажмите **"Старт"**

## ⚙️ Для второго клиента (распределённо)

На **другой машине** (в сети):
```bash
cd PiClient
dotnet run
```

Введите IP сервера (вместо localhost):
```
URL: http://192.168.1.100:5000
```

В браузере сервера вы увидите обоих клиентов в списке!

## 🔄 Параметры вычислений

В браузере есть кнопка **"🔄 Начать заново"**

Можно изменить:
- **Всего итераций**: количество членов ряда (по умолчанию 1 млрд)
- **Размер блока**: сколько итераций в одной задаче

## ❓ Если что-то не работает

### "Сервер не запускается"
Убедитесь, что установлен .NET 8.0 SDK:
```bash
dotnet --version
```
Должно быть: `8.0.xxx` или выше

### "Браузер говорит 'Failed to fetch'"
- Проверьте, что сервер запущен (`Ожидание клиентов...`)
- Проверьте URL: `http://localhost:5000` (не `https`)

### "WinForms клиент не подключается"
- Убедитесь, что сервер запущен
- Проверьте URL сервера в WinForms
- Если на разных машинах - используйте IP вместо localhost

### "График не обновляется"
- Дождитесь, пока клиент обработает несколько блоков (~10 сек)
- F5 в браузере для обновления

## 📱 Работает везде

Сервер слушает на **0.0.0.0:5000** - значит доступен:
- ✅ Локально: `http://localhost:5000`
- ✅ По IP: `http://192.168.1.100:5000`
- ✅ Из других ПК в сети
- ✅ Мобильных браузерах (адаптивный дизайн)
