# Распределённое вычисление числа π

## Описание проекта

Клиент-серверное приложение для параллельного и распределённого расчёта числовой константы π с использованием протокола HTTP. Проект состоит из двух компонентов:

- **PiServer** — сервер-координатор на ASP.NET Core с веб-интерфейсом в браузере
  - REST API для распределения задач
  - Веб-дашборд с графиком сходимости π в реал-тайме
  - Отслеживание активных клиентов
- **PiClient** — клиент с GUI (WinForms), выполняющий вычисления на локальной машине

## Веб-интерфейс

Откройте в браузере: **`http://localhost:5000`**

Дашборд показывает:
- 📊 **График сходимости π** — две кривые (Лейбниц и Нилаканта) сходятся к эталонному значению
- 📈 **Прогресс** — процент завершения в реал-тайме
- 🖥️ **Список активных клиентов** — какие машины вычисляют, сколько блоков обработано
- 📋 **Лог сервера** — все события вычислений
- 🎛️ **Управление** — кнопка для перезагрузки с новыми параметрами

## API сервера

### GET `/`
Веб-интерфейс (HTML + CSS + JavaScript).

### GET `/api/task`
Получить следующий блок для вычисления.
- **Ответ**: `204 No Content` (нет больше блоков) или `200 OK` с телом:
```json
{
  "startIndex": 0,
  "count": 10000000
}
```

### POST `/api/result`
Отправить результат вычисления.
```json
{
  "clientId": "Client-PC-NAME",
  "startIndex": 0,
  "count": 10000000,
  "leibnizPartialSum": 0.785398...,
  "nilakanthaPartialSum": 0.141592...,
  "elapsedMs": 1234.56
}
```

### GET `/api/status`
Получить текущий статус вычислений.
```json
{
  "totalChunks": 100,
  "completedChunks": 42,
  "progress": 42.0,
  "pi1": 3.14159...,
  "pi2": 3.14159...,
  "isComplete": false
}
```

### GET `/api/clients`
Получить список активных клиентов.
```json
[
  {
    "clientId": "Client-PC-NAME",
    "blockCount": 5,
    "totalTime": 1234.56,
    "lastUpdate": "2026-05-19T10:30:45"
  }
]
```

### POST `/api/reset`
Сбросить вычисления и начать заново.
```json
{
  "totalIterations": 1000000000,
  "chunkSize": 10000000
}
```

## Использование

### Требования
- .NET SDK 8.0 или выше (для разработки)
- .NET Runtime 8.0 или выше (для запуска)
- Windows 7/10/11 (для WinForms клиента)

### Запуск сервера

**Вариант 1: Через `dotnet run` (разработка)**
```bash
cd PiServer
dotnet run
```
Сервер запустится на `http://localhost:5000`

**Вариант 2: Запуск собранного exe**
```bash
cd PiServer/bin/Debug/net8.0
PiServer.exe
```

### Запуск клиента (WinForms)

```bash
cd PiClient
dotnet run
```

1. Введите URL сервера: `http://localhost:5000` (или IP машины сервера)
2. Нажмите кнопку "Старт"
3. В браузере откройте дашборд: `http://localhost:5000`

### Запуск на нескольких машинах

1. **На главной машине** — запустить сервер:
   ```bash
   cd PiServer && dotnet run
   ```

2. **На других машинах** — запустить клиентов с IP сервера:
   ```bash
   cd PiClient && dotnet run
   ```
   Введите адрес: `http://<IP_сервера>:5000`

3. **Мониторинг** — откройте в браузере `http://localhost:5000` на машине с сервером

## Вычислительные формулы

### Ряд Лейбница
$$\frac{\pi}{4} = \sum_{k=0}^{\infty} \frac{(-1)^k}{2k+1}$$

Медленная сходимость, но просто реализуется.

### Ряд Нилаканты
$$\pi = 3 + \sum_{k=1}^{\infty} \frac{(-1)^{k+1} \cdot 4}{2k(2k+1)(2k+2)}$$

Быстрее сходится, чем Лейбница.

## Интеграция с OpenServer

OpenServer — веб-сервер для Windows, обычно используется для PHP. Есть несколько способов работы:

### Способ 1: Запуск как приложение (рекомендуется)
1. Скопировать папку `PiServer` в OpenServer
2. Запустить: `PiServer/bin/Debug/net8.0/PiServer.exe`
3. Сервер будет доступен по адресу `http://localhost:5000` (или другому, если изменить в коде)

### Способ 2: PHP обёртка через OpenServer
Если нужна интеграция непосредственно в OpenServer (порт 80):
1. Создать файл `PiServer.php` в корне OpenServer
2. Перенаправлять запросы на `http://localhost:5000`
3. OpenServer будет работать как reverse proxy

**Важно**: ASP.NET Core сервер работает независимо и не требует OpenServer, но может быть запущен параллельно.

## Текущие ограничения и недостатки

### ✅ Реализовано

- ✅ Веб-интерфейс с HTML + CSS + JavaScript
- ✅ Реал-тайм график сходимости π
- ✅ Список активных клиентов
- ✅ Управление параметрами вычислений из браузера

### ⚠️ Технические замечания

- **Масштабируемость**: Сервер однопоточный (lock'и), может стать узким местом при 50+ клиентах одновременно
- **WebSocket**: Используется polling (каждые 1 сек), WebSocket был бы эффективнее
- **Браузер**: Требует поддержку ES6+ (работает во всех современных браузерах)
- **OpenServer**: ASP.NET Core — отдельный сервер, не встраивается в OpenServer как модуль

## Возможные улучшения

### 📌 WebSocket вместо polling
Текущее: опрос каждые 1 сек (много запросов)  
Улучшение: двусторонний WebSocket для инстантных обновлений

### 📌 Веб-клиент вместо WinForms
Текущее: требует .NET Runtime на Windows  
Улучшение: веб-приложение на HTML/JS, работает везде

### 📌 Экспорт результатов
Добавить скачивание графика и результатов в CSV/JSON

### 📌 Параллелизм на сервере
Текущее: lock'и блокируют при 50+ клиентах  
Улучшение: async/await вместо lock'ов, параллельная обработка результатов
