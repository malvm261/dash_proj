# Распределённое вычисление числа π

Учебный проект (Б23-181-1, прикладная математика, «Технологии программирования»):
приложение для **распределённого** расчёта числа π по сети с использованием HTTP.
Чем больше машин подключено к расчёту, тем быстрее считается итоговый результат —
каждая машина обрабатывает свой собственный диапазон членов ряда, а не дублирует
работу остальных.

## Состав проекта

- **PiWorker** — консольный HTTP-сервер на ASP.NET Core (Minimal API).
  Получает от диспетчера диапазон индексов членов ряда, считает по нему частичные
  суммы рядов Лейбница и Нилаканты, используя все ядра процессора (`Parallel.For`),
  и отдаёт текущий статус и частичный результат по запросу.
- **PiDispatcher** — GUI-приложение на Windows Forms. Делит общий объём вычислений
  на непересекающиеся диапазоны, раздаёт их воркерам, опрашивает их статус и
  складывает частичные суммы в одно общее значение π.

## Архитектура

```
                         HTTP (диапазон задания / статус)
┌────────────────────┐  ───────────────────────────────►  ┌────────────────────┐
│   PiDispatcher     │                                     │   PiWorker №1      │
│   (Windows Forms)  │  ◄───────────────────────────────  │ http://ПК1:5050    │
│                    │                                     └────────────────────┘
│  - делит диапазон  │  ───────────────────────────────►  ┌────────────────────┐
│    [0..N) на части │                                     │   PiWorker №2      │
│  - рассылает их    │  ◄───────────────────────────────  │ http://ПК2:5050    │
│  - опрашивает      │                                     └────────────────────┘
│    статус          │  ───────────────────────────────►  ┌────────────────────┐
│  - суммирует       │                                     │   PiWorker №3      │
│    результаты      │  ◄───────────────────────────────  │ http://ПК3:5050    │
└────────────────────┘                                     └────────────────────┘
```

PiDispatcher запускается на одной (управляющей) машине, а PiWorker — на каждой
машине, которая будет участвовать в расчёте (включая, при желании, ту же машину,
где работает диспетчер).

## Требования

- [.NET SDK 8.0](https://dotnet.microsoft.com/download/dotnet/8.0) (или новее — среда
  поддерживает запуск приложений `net8.0` через roll-forward) на каждой машине,
  где нужно **собирать** проект.
- Для **запуска** опубликованных самодостаточных (`self-contained`) exe-файлов
  установка .NET не требуется — рантайм уже включён в exe.
- PiWorker — кроссплатформенный (Windows / Linux / macOS).
- PiDispatcher — только Windows (Windows Forms).

## Вариант 1. Быстрый запуск через `dotnet run` (для разработки)

Подходит, когда на машине уже установлен .NET SDK — не нужно ничего публиковать,
удобно для тестирования на одном или нескольких компьютерах в локальной сети.

### Запуск воркера (на каждом ПК-вычислителе)

```bash
cd PiWorker
dotnet run
```

После запуска воркер поднимет HTTP-сервер на `http://0.0.0.0:5050` и будет ждать
задание от диспетчера. В консоли будет видно количество ядер CPU, прогресс и
частичные результаты.

> Чтобы узнать IP-адрес машины с воркером (его нужно будет ввести в диспетчере):
> - Windows: `ipconfig`
> - Linux/macOS: `ip addr` или `ifconfig`

### Запуск диспетчера (на управляющем ПК с Windows)

```powershell
cd PiDispatcher
dotnet run
```

Откроется окно GUI, в котором нужно:
1. Ввести адреса всех воркеров (по одному в строке), например:
   ```
   http://192.168.1.10:5050
   http://192.168.1.11:5050
   http://localhost:5050
   ```
2. Указать общее число членов ряда и интервал опроса статуса.
3. Нажать **«Старт»** — диспетчер разобьёт диапазон поровну между воркерами,
   разошлёт задания и будет показывать прогресс и общий результат π в таблице.

## Вариант 2. Публикация самодостаточных exe-файлов (для раздачи без установки .NET)

`dotnet publish` со флагом `--self-contained true` собирает исполняемый файл,
который включает в себя весь нужный рантайм .NET — такой exe можно скопировать
на любую машину той же ОС/архитектуры и запустить без установки .NET SDK.

Общий шаблон команды:

```bash
dotnet publish <путь_к_проекту> -c Release -r <RID> --self-contained true -p:PublishSingleFile=true -o <папка_вывода>
```

где `<RID>` — Runtime Identifier целевой системы:

| Система                       | RID            |
|-------------------------------|----------------|
| Windows 64-бит                | `win-x64`      |
| Windows ARM64                 | `win-arm64`    |
| Linux 64-бит (x86_64)         | `linux-x64`    |
| Linux ARM64                   | `linux-arm64`  |
| macOS Intel                   | `osx-x64`      |
| macOS Apple Silicon (M1/M2/…) | `osx-arm64`    |

### 2.1. Сборка PiWorker под Windows

```powershell
dotnet publish PiWorker/PiWorker.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o publish-worker-win
```

Результат: `publish-worker-win/PiWorker.exe`. Копируем папку на целевой ПК и
запускаем `PiWorker.exe` (двойным щелчком или из консоли).

### 2.2. Сборка PiWorker под Linux

```bash
dotnet publish PiWorker/PiWorker.csproj -c Release -r linux-x64 --self-contained true -p:PublishSingleFile=true -o publish-worker-linux
```

Результат: `publish-worker-linux/PiWorker`. На целевой Linux-машине:

```bash
chmod +x PiWorker      # дать файлу права на выполнение (один раз)
./PiWorker
```

### 2.3. Сборка PiWorker под macOS

```bash
# Apple Silicon (M1/M2/…):
dotnet publish PiWorker/PiWorker.csproj -c Release -r osx-arm64 --self-contained true -p:PublishSingleFile=true -o publish-worker-osx

# Intel:
dotnet publish PiWorker/PiWorker.csproj -c Release -r osx-x64 --self-contained true -p:PublishSingleFile=true -o publish-worker-osx
```

Результат: `publish-worker-osx/PiWorker`. Запуск аналогично Linux:

```bash
chmod +x PiWorker
./PiWorker
```

### 2.4. Сборка PiDispatcher (Windows GUI)

PiDispatcher — это Windows Forms приложение, поэтому собирать и публиковать его
имеет смысл только под Windows-рантайм (`win-x64` / `win-arm64`). Команда работает
как на Windows, так и при кросс-компиляции с Linux/macOS — в `.csproj` уже включена
опция `<EnableWindowsTargeting>true</EnableWindowsTargeting>`:

```bash
dotnet publish PiDispatcher/PiDispatcher.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o publish-dispatcher-win
```

Результат: `publish-dispatcher-win/PiDispatcher.exe`. Копируем папку на Windows-ПК
и запускаем `PiDispatcher.exe` двойным щелчком — окно GUI откроется сразу,
установка .NET не требуется.

> Если публикация выполняется не на Windows (например, на macOS), убедитесь, что
> в `PiDispatcher.csproj` присутствует строка `<EnableWindowsTargeting>true</EnableWindowsTargeting>` —
> без неё сборка завершится ошибкой `NETSDK1100`.

## Сборка через Visual Studio (Windows)

1. Откройте `PiDistributed.sln` в Visual Studio.
2. В Solution Explorer выберите нужный проект (`PiWorker` или `PiDispatcher`),
   щёлкните правой кнопкой → **Publish…**.
3. В качестве цели выберите **Folder**, задайте Target Runtime (`win-x64`,
   `linux-x64` и т. д.), включите **Produce single file** и **Self-contained**.
4. Нажмите **Publish** — exe-файл появится в указанной папке.

Либо просто нажмите **Build → Build Solution** (`Ctrl+Shift+B`) для обычной
отладочной сборки — DLL появятся в `PiWorker/bin/Debug/net8.0/` и
`PiDispatcher/bin/Debug/net8.0-windows/`, запускать их нужно через `dotnet <имя>.dll`
или из Visual Studio (F5).

## Порядок запуска расчёта

1. На каждой машине-вычислителе запустите `PiWorker` (через `dotnet run` или
   опубликованный exe). Каждый воркер слушает порт **5050** и выводит свой IP
   и количество доступных ядер CPU.
2. На управляющей Windows-машине запустите `PiDispatcher`.
3. В поле «Адреса воркеров» перечислите адреса всех запущенных воркеров —
   каждый на отдельной строке, в формате `http://<IP-адрес>:5050`.
4. Укажите общее число членов ряда и интервал опроса статуса (мс).
5. Нажмите **«Старт»**. Диспетчер автоматически:
   - разобьёт общий диапазон `[0; N)` на примерно равные непересекающиеся куски —
     по одному на каждый воркер;
   - разошлёт каждому воркеру именно его диапазон (`POST /api/start`);
   - будет периодически опрашивать статус каждого воркера (`GET /api/status`)
     и обновлять таблицу с прогрессом;
   - сложит частичные суммы рядов Лейбница и Нилаканты со всех воркеров в одно
     общее приближение числа π и покажет его рядом с эталонным значением.
6. Кнопка **«Стоп»** останавливает расчёт на всех воркерах (`POST /api/stop`).

Поскольку каждый воркер обрабатывает свой собственный, не пересекающийся с
другими диапазон членов ряда, добавление новых машин действительно ускоряет
получение итогового результата — это и есть распределённое вычисление, а не
просто параллельное дублирование одной и той же работы.

## HTTP API воркера (PiWorker)

| Метод | Путь           | Назначение                                                |
|-------|----------------|-----------------------------------------------------------|
| POST  | `/api/start`   | Назначить воркеру диапазон `{ "RangeStart": ..., "RangeCount": ... }` и запустить расчёт |
| GET   | `/api/status`  | Получить текущий статус: прогресс, число посчитанных членов, частичные суммы по обеим формулам |
| POST  | `/api/stop`    | Остановить текущий расчёт                                  |

По умолчанию воркер слушает `http://0.0.0.0:5050` (доступен по любому сетевому
адресу машины на этом порту).

## Используемые формулы

- **Ряд Лейбница**: π = 4 · Σ (-1)ᵏ ⁄ (2k+1), k = 0, 1, 2, …
- **Ряд Нилаканты (Мадхавы)**: π = 3 + Σ (-1)ⁿ⁺¹ · 4 ⁄ (2n·(2n+1)·(2n+2)), n = 1, 2, 3, …
